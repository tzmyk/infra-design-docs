# コスト見積もり・最適化設計

> **前提:** 月間注文数 10 万件、ピーク 100 req/sec  
> **見積もり基準:** 2026年4月時点の東京リージョン料金

## コスト最適化の優先順位

### すぐやるべき（設計段階で組み込む）

| 施策 | 削減額/月 | 実施コスト |
|---|---|---|
| S3 ゲートウェイ型 VPC エンドポイント | $5〜10 | 低（Terraform 数行） |
| Lambda arm64 (Graviton2) に変更 | $0.6（20%削減） | 低（runtime 変更のみ） |
| Aurora Serverless v2 に変更（開発環境） | $60 → $10 | 中（設定変更） |
| CloudWatch Logs の保持期間設定 | $1〜3 | 低 |

### 規模が大きくなったら検討

| 施策 | 条件 | 効果 |
|---|---|---|
| Compute Savings Plans | Lambda 使用量が安定したら | 最大 17% 削減 |
| Aurora リザーブドインスタンス | 1年以上継続が確定したら | 最大 43% 削減 |
| CloudFront + S3 | 領収書 PDF のアクセスが多いなら | データ転送費削減 |

## 環境別コスト設計

本番と開発で同じ構成を使わない。

| リソース | 本番 (prod) | 開発 (dev) |
|---|---|---|
| Aurora | db.t4g.medium × 2 | Aurora Serverless v2（最小 0.5 ACU） |
| ECS | 常時 2 タスク | 0（スケジュールで夜間停止） |
| NAT Gateway | 常時起動 | 常時起動（削れるが管理コスト増） |
| WAF | 有効 | **無効**（$25 節約） |
| Secrets Manager | 本番シークレット | 開発用シークレット（同額だが分離） |

> **dev 環境の概算:** $80〜100/月（本番の 35〜40%）

## コスト異常検知

```
設定するアラート:

1. 月次予算アラート（AWS Budgets）
   - 予算: $300/月
   - 80% 到達で通知: $240
   - 100% 到達で通知: $300

2. 日次コストスパイク（Cost Anomaly Detection）
   - 閾値: 前日比 +$20 以上
   - 通知先: SNS → Slack
```

> **よくある異常コストの原因:**  
> - DDoS によるリクエスト急増 → WAF で対策済み  
> - Lambda の無限ループ → タイムアウト設定で上限を設ける  
> - S3 Glacier からの大量 Restore → ライフサイクルポリシーを確認

---

## 設計上の禁止事項（AI 参照用）

- 開発環境に本番と同じインスタンスサイズを使わない
- NAT Gateway のログを有効にしない（課金が跳ね上がる）
- CloudWatch Logs の保持期間を「失効なし」に設定しない（必ず期間を設定）
- S3 バケットに Requester Pays を設定しない（意図しない課金が発生する）
