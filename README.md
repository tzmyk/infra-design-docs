# インフラ設計書セット（AWS）

インフラ研修用・GitHub Copilot 対応設計書テンプレート。

## フォーマット方針

| 用途 | 形式 | 理由 |
|---|---|---|
| 概要・説明文 | Markdown + 表 | 人間がそのまま読める |
| ネットワーク図・フロー | Mermaid | テキストで書けて AI も構造を読める |
| リソース定義（機械処理用） | Markdown 内の YAML コードブロック | 設計書の文脈の中に収める |
| 禁止事項・設計判断の根拠 | Markdown の箇条書き | AIへの制約として機能する |

**基本ルール: YAML ファイル単体では作らない。Markdown の中に埋め込む。**

## 構成

```
docs/         設計書（期待値の定義）
  01_network/   VPC・サブネット・ルーティング・セキュリティグループ
  02_security/  セキュリティ設計・暗号化・証跡
  03_iam/       IAMロール・ポリシー・最小権限設計
  04_compute/   Lambda・ECS・EC2などコンピューティング設計
  05_cost/      コスト見積もり・最適化方針
  06_operations/ デプロイ・監視・インシデント対応

tests/        インフラテスト（設計書の期待値を実 AWS で検証）
  README.md   テスト方針・テストケース一覧
```

`docs/` が「あるべき姿」、`tests/` が「あるべき姿になっているかの検証」という関係。

## AI への渡し方

```
# Copilot / Claude へのプロンプト例

以下の設計書を読んで、Terraform のリソースを生成してください。

@docs/01_network/vpc-design.md  ← まず渡す
@docs/03_iam/roles.md           ← 対象コンポーネントの IAM を渡す

「## Terraform 定義」セクションのコードブロックを参考に、
同じスタイルで <対象リソース名> を実装してください。
```
